# Facial Emotion Detection System - Code Execution & Analysis Report

## 1. Overview
The Facial Emotion Detection System is designed to classify human emotions from facial images into six distinct categories: Anger, Fear, Happy, Neutral, Sad, and Surprise. The system processes grayscale images, trains a Deep Convolutional Neural Network (CNN) using TensorFlow/Keras, and features a real-time tracking script using Haar Cascades and the trained model.

## 2. Environment Setup
The environment relies on the following key libraries: `pandas`, `numpy`, `cv2` (OpenCV), `tensorflow.keras`, `sklearn`, and `matplotlib`. GPU acceleration was checked, and the outputs log indicated `Num GPUs Available: 0`, meaning training fallback was directed to the CPU.

## 3. Data Preprocessing
Images are loaded individually from an organized directory structure containing class-labeled folders. The data pipeline involves:
- Converting images to grayscale and resizing them to uniformly be 48x48 pixels.
- Normalizing pixel values to the range [0, 1] (dividing by 255.0).
- Expanding dimensions to match the required CNN input shape of `(48, 48, 1)`.
- One-hot encoding the categorical labels to suit categorical crossentropy loss.
- Splitting the complete dataset into a training (80%) and validation (20%) set using stratified sampling to preserve the existing class distribution.

## 4. Model Architecture
The system uses a custom Convolutional Neural Network (CNN) configured as a Keras Sequential model. It consists of four main Convolutional blocks followed by fully connected dense layers.
- **Convolutional Layers:** Four layers with increasing filters (64, 128, 512, 512) applying ReLU activations to extract spatial hierarchies of features.
- **Pooling:** MaxPooling2D layers reduce spatial dimensions systematically (down to 3x3 before flattening).
- **Regularization:** Batch Normalization is aggressively applied after each Conv2D and Dense layer to stabilize and accelerate training. Dropout layers (rates of 0.25 in Conv blocks, and 0.5 in Dense blocks) are utilized to prevent overfitting the training data.
- **Classifier Layer:** The network flattens the extracted features and feeds them into Dense layers (256, 512 nodes), culminating in a final softmax output layer with 6 nodes (for the 6 emotion classes).

**Model Summary Execution Outputs:**
- Total parameters: 4,478,214 (17.08 MB)
- Trainable parameters: 4,474,246 (17.07 MB)
- Non-trainable parameters: 3,968 (15.50 KB) (due to Batch Normalization moving metrics)
- Optimizer: Adam with a learning rate of 0.0001
- Loss function: Categorical Crossentropy

## 5. Model Training Configuration
The training procedure implements several best industry practices to optimize performance and handle dataset class imbalances:
- **Class Weights:** Computed using `sklearn` to give balanced importance to underrepresented emotions implicitly.
- **Data Augmentation:** Utilizes `ImageDataGenerator` configured for rotation (10 degrees), width/height shifts (0.1), zoom (0.1), and horizontal flipping to artificially expand the training dataset and improve generalization to unseen faces.
- **Callbacks:**
  - `ModelCheckpoint`: Saves exclusively the best model weights based on validation accuracy (`emotion_model_nb.keras`).
  - `ReduceLROnPlateau`: Reduces the learning rate by a factor of 0.2 if validation loss plateaus for 5 epochs.
  - `EarlyStopping`: Aborts training if validation loss does not improve for 15 consecutive epochs, restoring the optimal model weights automatically.
- **Hyperparameters:** Batch size of 64, scheduled to iterate up to 100 epochs.

## 6. Evaluation Execution Metrics
The notebook visualizes the model performance history by plotting the training versus validation accuracy, alongside training versus validation loss (as confirmed via the plotted output images in cell 12). 

## 7. Real-Time Detection
The final block of the notebook implements a real-time facial emotion recognition overlay:
- Captures live video feed using OpenCV (`cv2.VideoCapture(0)`).
- Detects global face coordinates in the current frame using OpenCV Haar Cascades (`haarcascade_frontalface_default.xml`).
- Crops the detected Region of Interest (ROI), resizes it to 48x48, normalizes it, and feeds it into the loaded `.keras` prediction model.
- Analyzes the probability array to extract the maximum confidence index, resolving the string label natively.
- Annotates the raw video frame with a bounding box, the predicted emotion, and the confidence score.

## Conclusion
The script successfully develops an end-to-end pipeline: from organizing raw directory-based pixel data to training a heavily fortified CNN framework mapping complex non-linear facial expressions, resulting in a deployable tool capable of high-throughput real-time webcam inference.
